Berechtigungen

API Keys werden im CMS erstellt und können pro App sowie als Read-only oder Write-Key begrenzt werden.

Read-only

Darf nur lesen: `GET` auf Listen- und Detail-Endpunkte.

Lesen und schreiben

Darf zusätzlich `POST`, `PATCH`, `PUT` und `DELETE` ausführen.

Ping

Prüft, ob der API-Key gültig ist und die Developer API erreichbar ist.

GET

{{ api_ping_url }}

Antwortet nur mit gültiger Authentifizierung. Ein Read-only Key reicht aus.

GET {{ api_ping_url }}
Authorization: Bearer <api-key>
{
  "ok": true,
  "message": "YooLink Developer API ist erreichbar und authentifiziert.",
  "authenticated": true,
  "user": "owner",
  "access_level": "read",
  "allowed_apps": ["blog"]
}

YooLink Connect

Browserbasierter Verbindungsflow fuer externe Plattformen. Der Nutzer loggt sich im jeweiligen YooLink CMS ein und die Plattform tauscht danach einen kurzlebigen Code gegen einen API-Key.

GET

{{ api_connect_authorize_url }}

Startet den Login- und Consent-Flow im Browser. Parameter: `client_name`, `redirect_uri`, `scope`, `access_level`, `state`, `code_challenge`, `code_challenge_method=S256`.

POST

{{ api_connect_token_url }}

Tauscht `code`, `code_verifier` und `redirect_uri` gegen einen Developer API Key. Der Key wird nur einmal in dieser Antwort ausgegeben.

{
  "grant_type": "authorization_code",
  "code": "yl_connect_...",
  "code_verifier": "pkce-verifier-from-your-platform",
  "redirect_uri": "https://ki-plattform.example.com/api/yoolink/callback"
}

Blogs

Externe Systeme können Blogartikel lesen, erstellen, aktualisieren und löschen.

App Scope: blog
GET

/api/cms/blog/

Listet Blogs als kompakte Übersicht ohne `body`, `code` und `translations`. Optionale Query-Parameter: `q`, `language`, `active`, `original_only`.

GET /api/cms/blog/?language=de&active=true&original_only=true
GET

/api/cms/blog/{id}/

Liest einen einzelnen Blog inklusive `absolute_url`, `title_image_url`, `code` und `translations`. In `translations` stehen Original und Sprachvarianten mit API- und Public-Link.

POST

/api/cms/blog/

Erstellt einen Blog. Erfordert einen Write-Key. Als JSON für reine Textblogs oder mit `title_image_media_id`, wenn vorher ein Bild über `/api/cms/blog/media/` hochgeladen wurde. Alternativ als Multipart-Formdaten, wenn direkt ein `title_image` mitgesendet wird.

{
  "title": "Event Rückblick",
  "description": "Kurzer SEO-Teaser für Blogkarten und Meta Description.",
    "markdown": "## Rückblick\n\nHTML-freier Markdown-Inhalt des automatisch generierten Blogartikels.",
  "title_image_media_id": 44,
  "active": true,
  "language": "de"
}
POST

/api/cms/blog/media/

Lädt ein Content-Bild hoch und gibt URL, Markdown-Snippet und HTML-Snippet zurück. Erfordert einen Write-Key. In Postman als `form-data` senden.

file: event.png
title: Event Bild
alt_text: Volles Haus beim Event
PATCH

/api/cms/blog/{id}/

Aktualisiert einzelne Felder. Erfordert einen Write-Key.

{
  "active": false,
  "description": "Aktualisierte Kurzbeschreibung."
}
DELETE

/api/cms/blog/{id}/

Löscht einen Blog. Erfordert einen Write-Key.

Blog Felder

Die API nimmt JSON oder Multipart-Formdaten an. Für JSON-Workflows erst Medien hochladen und dann per ID referenzieren.

Feld Typ Pflicht Beschreibung
titlestringjaBlogtitel. Der Slug wird automatisch erzeugt.
descriptionstringjaKurzbeschreibung für Blogkarten und SEO.
markdownstringempfohlenPrimäre Textquelle für KI-Workflows. Wird zu HTML gerendert und für den Builder in editierbare Blöcke umgewandelt.
bodyHTML stringbedingtKompatibilitätsfeld für gerenderten Bloginhalt. Wenn kein `markdown` gesendet wird, erzeugt die API Markdown daraus.
codearray/objectbedingtInternes Blog-Builder-JSON. Wenn kein `body` gesendet wird, rendert die API daraus HTML.
activebooleanneinSteuert, ob der Blog öffentlich sichtbar ist.
languagestringneinZum Beispiel `de` oder `en`.
originalinteger/nullneinID des Original-Blogs, wenn eine Sprachvariante erstellt wird. Pro Original kann jede Sprache nur einmal vorkommen.
title_imagefileneinTitelbild per Multipart-Upload.
title_image_media_idintegerneinID aus `POST /api/cms/blog/media/`, um ein zuvor hochgeladenes Bild als Titelbild zu verwenden. Für JSON-Requests statt `title_image` nutzen.

Status Codes

200

Erfolgreich gelesen oder aktualisiert.

201

Blog wurde erstellt.

401

Key fehlt, ist ungültig oder abgelaufen.

403

Key hat keinen passenden Scope oder keine Schreibrechte.
